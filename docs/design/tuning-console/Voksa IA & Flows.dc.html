// hard behavioral contracts (the design must not fight these)
⤢ widen, never clamp — a loaded value outside a slider's range widens min/max and relaxes step to any; the readout always shows the true value.
load = replace — absent keys fall back to engine defaults; loading over a dirty session must equal what voksa --config would play.
Δ delta-only export — attitudinals/phonemes export only f32-different values (fround compare); hence modified indicators everywhere.
defaults from the engine — sliders seed from voksa_default_params(); UI equals engine by construction.
flat disables voice shaping — all 465 controls + the xu flag grey out; text, playback and share stay live.
gesture-gated audio — first ▶ creates the AudioContext; re-speaks replace the live node, never stack; ~400 ms debounce collapses drags.
// the voice-table pattern — why grid → editor
∴377 sliders as collapsible rows (the old page) buries state: you can't see where a tuning deviates. The phoneme grid is a map — 27 keycaps grouped by manner class, each carrying an ember Δ dot — so a loaded config is legible at a glance.
∴One phoneme in focus keeps ≤24 controls on screen. The 3×3 formant matrix (F1–F3 × freq/bw/amp) mirrors how a Klatt voice is actually thought about; stops get closure + burst matrices + timing; diphthongs/[h] expose only their one free parameter, with a note explaining why.
∴changed-only dims untouched cells (never hides — grid stays spatially stable); per-phoneme ↺ and table-level ↺ match the delta-export mental model.
∴Phone: the same grid wraps above the editor — navigation survives 360 px because cells are 34 px touch targets, not table rows.
